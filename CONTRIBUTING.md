# Contributing

Thanks for considering a contribution.

## Setup

1. Install dependencies:

```bash
composer install
```

2. Run quality checks before opening a PR:

```bash
composer format-check
composer analyse
composer refactor
composer test
```

## Pull Requests

- Keep changes focused and minimal.
- Include tests for bug fixes and behavior changes.
- Update docs when public behavior or APIs change.

## Commit Style

Use clear imperative commit messages that explain intent.

## Security

Please do not report security issues in public issue threads. See `SECURITY.md`.
